Part A: Data Processing¶

DNA sequences are analogous to parsing text data for they are both sequential in nature. With a 3mer-based reading frame, 3 DNA neucleotides can be binned to correspond mRNA code which serves as the basis for amino acids, coding for functional proteins. For each DNA sample, there are three possible open reading frames. In order to capture all reading frames, I transformed DNA sequences into a series of "words" by binning each of the three DNA sequences as mRNA by going through each seqeunce. To increase continuity of DNA sequence patterns, I then stacked mRNA code in sequence on top of each other. To keep each sample the same length, I used the shortest length out of all genes as a limit to trim the end of each genetic sequence.

To translate each mRNA code into numerical vector, I applied one-hot-encoding to the 64 possible 3mers. Thus, a single gene sample's mRNA code corresponds to a 64 X 891 2D numeric numpy array. After concactnating two nearest sequential mRNA codes, we will have a 128 X 891 2D numeric matrix per sample. Trnasforming each of the 1000 gene samples randomly selected from the previous section stratified by resistance type, the final numpy array results in a 3D matrix of shape 1000 X 128 X 891.

Advantages of this method include:

  1. Thoroughly accounting for each possible transcription outcomes (mRNAs)
  2. Increasing the continuity of pattern in each genetic sample

(reference: "DNA Sequence Classification by Convolutional Neural Network" Nguyen et al. https://www.scirp.org/pdf/JBiSE_2016042713533805.pdf)

##Define a function to process all samples by parallel processing

def process(index):
    
    ##Sample from down_sized pool
    s=down_x[index]
    
    ##Trim length to match the shortest length of the entire data
    if len(s)> len(min(xs, key=len)):
        s=s[0:len(min(xs, key=len) )]
    
    ##Find all possible open reading frame 
    group=[s[i:i+3] for i in range(0, len(s))]
    
    ##Concactnate with the next closest word (mRNA)
    long=[]
    for i in range(len(group)-1):
        if len(group[i])==3 & len(group[i+1])==3:
            g1=group[i]
            g2=group[i+1]
            bind=np.concatenate((encode[codon.index[codon['code']==g1][0]], 
                                 encode[codon.index[codon['code']==g2][0]]))
            long.append(bind)
    longstack=np.column_stack(long)
    return (longstack)

Part B: Model construction and training¶

According to a recent paper "NCNet: Deep Learning Network Models for Predicting Function of Non-coding DNA" by Zhang et al., a hybrid framework integrating a Convolutional Neural Network (CNN) and bi-directional LSTM Recurrent Neural Network (RNN) are accurate in predicting the noncoding function from a sequence. This hybrid model takes advantage of the DNA sequence parsed as text data processed above (3D numpy array) and used one 1D convolutional layer followed by a max pooling to comprehensively learn features of the dataset. Then using the bi-directional LSTM RNN further trains the model to recognize the temporal nature of the DNA reads. The combinations of such model are plausible to reach optimal performance in predicting DNA function, which is fitting for the purpose of this project.

The deployment of a bi-directional LSTM RNN fits a biological context. Becuase one DNA strand has 6 open reading frames, three on the forward strand and three on the backward strand, the bi-directional structure allowed the model to learn the input sequence in a forward timestemp and backward timestemp. Thus, I believe that a bidirectional structure can outperform a single direction LSTM for DNA sequences.

A graphical presentation of the model structure (Zhang et al.):

The convolutionary layer with ReLU activation was followed by maxpooling and a drop out to avoid overfitting. Maxpooling was followed by a bi-directional RNN LSTM layer with another drop out layer. Both forward and backward LSTM utilized Relu activation. Then, a flatten layer and two dense layer with a final activation function was set to sigmoid for the binary outcome. ADAM optimizer was used and the objective function was binary crossentropy.

Three key features were chosen to tune with 5-fold cross validation random-grid search to speed up computation. First, I tuned the the number of filters in the 1D CNN layer. I chose to determine kernel_size as 3 to mimic the translational process that reads 3 amino acids at once. Becuase the CNN and RNN are sensitive to the number of epochs and batch size trained, I also added these two parameters into the parameter grid.

##Model creation to CV with KerasClassifier

def create_model(num_filters=32):
    model=Sequential()

    ###1D CNN
    model.add(Conv1D(filters=num_filters, ###tune parameter
                     kernel_size=3, ###open reading frame
                     activation="relu",
                     input_shape=data_new.shape[1:] ) )
    model.add(MaxPooling1D(pool_size=3,strides=3)) 
    model.add(Dropout(0.2))

    ###Bidirectional RNN LSTM
    forward_layer = LSTM(32, 
                         activation='relu',
                         return_sequences=True)
    backward_layer = LSTM(32, 
                          activation='relu', 
                          return_sequences=True,
                          go_backwards=True)
    model.add(Bidirectional(forward_layer, 
                            backward_layer=backward_layer))
    model.add(Dropout(0.5))

    ###Flatten
    model.add(Flatten())

    ###Dense
    model.add(Dense(64))
    model.add(Activation('relu'))

    model.add(Dense(1))
    model.add(Activation('sigmoid'))

    ##Model compilation
    model.compile(loss="binary_crossentropy",
                  optimizer="adam", 
                  metrics=['accuracy'])
    return model

Part E: Conclusion and challenges:¶

A hybrid structure of 1D convolutionary network with bidirectional LSTM structures achieved high performances with a sequential mRNA one-hot-encoded data stacked with neareset information as an input. 1D CNN was chosen due to the 3D nature of the DNA sequencing data. A combination of CNN and RNN allowed the model to learn patterns of the DNA sequences by CNN. A bidirectional RNN further mimics the biological process of translating RNA sequences on a forward and backward strand. This proposed model was able to achieve a testing accuracy of 94% on the downsampled dataset.

Challenges for this project include a biological and a computational aspect:

  1. Computational resources: I trained the model on the Harvard Cannon cluster which utilized slurm schedulers. Due to high volumn of requests, I was not able to request high number of CPUs and memories for the task. Thus, this project could be deemed as a pilot run for a subset of the data. It would be more optimal if the random search could include more hyperparameters such as filter sizes for LSTM layers and drop out percentages. It would also be preferable to train on a larger set of data.
  2. As mentioned in the feature visualization section, it was challenging to identify underlying biological patterns learned by the filters in the CNN layer and the features learned in subsequent layers. This lack of interpretability can partially be attributed to the stacking structure of the mRNA codes in the data processing step which would require further mapping of the orignal DNA sequences. Another reason can be the lack of reference genome of which these gene sequences are from to allow mapping of known DNA motifs to the features learned at each layer.
